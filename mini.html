?왜 Code예요? · 채팅과 뭐가 달라요

채팅으로도 계산기·차트 하나는 만듭니다(아티팩트). 하지만 채팅 안 일회성입니다.
Code는 내 폴더·내 데이터를 직접 다루고, 진짜 파일(.docx·.png·.xlsx)을 만들어 남깁니다 — 여러 파일·여러 단계를, 프로젝트로 지속. 그래서 논문 한 편을 통째로 돌립니다.

3간단 코딩 — A′ 먼저 · 복붙

처음이면 A′부터(Python 불요·확실히 성공) — 되는 분은 B·C로. 끝나면 파일 탐색기를 열어 새로 생긴 진짜 파일을 확인하세요 — 채팅엔 없는 것. (모두 교육용 가상 데이터)

A′ · 내 폴더 읽어 정리 (가장 쉬움)

이 폴더의 파일들을 모두 읽고, 각 파일 한 줄 요약과 전체 목록을 INDEX.md로 만들어줘. (파일은 옮기지 말고 읽기만)

→ 진짜 INDEX.md 생성. 여러 파일을 한 번에 읽음(채팅엔 불가).

B · 내 데이터로 진짜 분석 (연구)

이 폴더의 sample.csv로 군별(group) 평균±표준편차 표와, 전후(before→after) 변화량 막대그래프(에러바)를 만들어줘. 표는 result.md, 그래프는 chart.png, 요약은 summary.xlsx로 저장. python 필요하면 설치해서 진행하고, 막히면 뭐가 막혔는지 알려줘.

→ 진짜 chart.png·summary.xlsx 생성. (채팅 아티팩트는 일회성 차트만)

C · 진짜 파일 변환·병합 (실무)

이 폴더의 notes_*.md 파일들을 하나의 Word(.docx)로 합쳐줘 — 각 파일명을 소제목으로. pandoc 없으면 python-docx로 진행. 완성 파일 경로 알려줘.

→ 진짜 .docx 생성. (채팅은 내 로컬 파일들을 못 합침)

📄 산출물 열기 — INDEX.md·result.md 같은 마크다운(.md)은 메모장으로도 바로 열립니다. 서식까지 깔끔히 보려면 무료 뷰어 Mark를 쓰세요: Mark-windows-x64.zip 다운로드 (Windows · 압축 풀어 실행). .docx·.xlsx·.png은 Word·Excel·사진 앱으로 열면 됩니다.

② 두뇌(brain)를 골라 쓴다

④ LLM wiki — 옵시디언 없이 지식 위키 (예: 인맥 노트)

에이전트가 Markdown 노트를 쌓고 연결해 나만의 지식베이스를 만듭니다(Karpathy LLM Wiki). 매번 새로 찾는 RAG가 아니라 쌓을수록 똑똑해지는 메모.

💡 예 — 인맥 노트: 친구마다 /people에 노트 하나(자녀 이름·나이·근황). "7월 생일인 친구 찾아줘" → 여러 문서를 가로질러 정리, "○○ 자녀가 몇 살이었지?" → 바로 답. 친구가 내 아이 이름·나이를 기억해 물어봐 주면 반갑듯, 그걸 대신 챙겨주는 셈 — 나이 들수록 이런 디테일이 관계의 자산.

빠른 시작 — Code에 이렇게:

이 폴더에 LLM wiki를 시작하자. people 폴더에 사람별 노트(이름·생일·자녀 이름/나이·근황)와 전체 index.md를 만들고 계속 갱신해줘. 먼저 구조부터.

제대로/오래 쓰려면 — Karpathy의 LLM wiki 패턴(공개 gist, "Code에 붙여넣어 쓰라"고 만든 스펙)대로 시키면 index.md(목차)+log.md(기록)로 일관되게 쌓입니다 — "이 gist대로 인맥 위키 만들어줘". 읽은 논문 정리에도 그대로. 옵시디언으로 열면 그래프로 보임 · 개인 노트이니 내 기기에서.
