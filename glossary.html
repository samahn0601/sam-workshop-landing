💡 대부분은 외울 필요 없습니다. 오늘 git·코딩·파이썬 같은 건 Claude(AI)가 알아서 하고, 여러분은 한국어로 말하고 확인만 합니다. 이 표는 "그게 무슨 말이지?" 싶을 때 보는 사전입니다.

🧩① 설치할 때 보이는 말

Claude(클로드)AI

오늘 함께 일하는 AI입니다. ChatGPT 같은 대화형 AI인데, 글쓰기뿐 아니라 파일·폴더를 직접 다룰 수 있습니다.

Claude Desktop 앱

컴퓨터에 설치하는 Claude 프로그램. 웹 대신 앱으로 쓰면 내 컴퓨터의 폴더를 다룰 수 있습니다.

Code 탭 오늘 주력

Claude 앱 위쪽의 탭 하나. 오늘 논문 작업을 하는 작업실입니다. 이름은 "Code(코드)"지만 여러분이 코딩하는 게 아니라, 여기서 Claude가 파일을 만들고 정리합니다.

Claude Code

위 Code 탭의 정식 이름. "파일·폴더·도구를 다룰 수 있는 Claude"라고 보면 됩니다.

코딩 / 프로그래밍 오늘 직접 안 함

컴퓨터에 시킬 일을 정해진 글(코드)로 적는 것. 오늘은 여러분이 코딩하지 않습니다 — Claude가 대신 합니다.

파이썬Python Claude가 설치

널리 쓰이는 프로그래밍 언어. 오늘은 참고문헌·통계를 점검하는 작은 프로그램이 파이썬으로 돌아갑니다. 없으면 Claude가 알아서 깔아줍니다.

터미널 / 명령어(CLI) 오늘 거의 안 씀

글자로 컴퓨터에 명령하는 검은 창. 옛날 방식이라 어렵지만, 오늘은 거의 안 씁니다 — Claude에게 한국어로 말하면 됩니다.

git(깃) Claude가 함

파일의 변경 이력을 기록하고 되돌릴 수 있게 해주는 도구. 문서의 "버전 관리·실행 취소"라고 보면 됩니다. Windows에선 이게 있어야 Code 탭이 처음 켜집니다.

GitHub(깃허브)

git 저장소를 인터넷에 올려두는 곳. "코드용 구글 드라이브"라고 생각하면 됩니다. 워크숍 도구를 여기서 받습니다.

저장소 / repo(레포)repository

한 프로젝트의 파일들을 묶어둔 폴더. 워크숍 도구 묶음이 samahn0601/sam-workshop이라는 저장소입니다.

clone(클론) / 다운로드

인터넷의 저장소를 내 컴퓨터로 복제해 가져오는 것. 오늘은 Claude가 하거나, ZIP을 받아 풀면 됩니다.

ZIP / 압축 풀기

여러 파일을 하나로 묶은 파일(.zip). 받아서 "압축 풀기"(Windows 우클릭 / Mac 더블클릭)하면 폴더가 생깁니다. 오늘 가장 쉬운 설치법.

폴더 / 파일 경로

파일이 들어 있는 위치. .claude/skills/처럼 점(.)으로 시작하는 폴더는 "설정용 숨김 폴더"입니다. 오늘 작업 폴더 하나를 고르면 거기에 결과가 쌓입니다.

skill(스킬)

특정 작업을 반복 가능하게 묶은 도구 프리셋. 예: 저널 찾기, 참고문헌 점검. 오늘 18개가 설치되며, 이름을 외울 필요 없이 "시작하자"라고 하면 알아서 불립니다.

발화 / 프롬프트prompt

Claude 입력창에 적어 보내는 말·지시. "붙여넣기 발화"는 미리 준비한 문장을 복사해 그대로 붙여넣는 것.

(새) 세션

대화를 새로 시작하는 것(새 채팅창). 방금 설치한 도구는 새 세션을 열어야 보입니다.

권한 승인permission

Claude가 파일을 만들거나 명령을 실행하기 전에 "해도 될까요?"라고 물을 때 누르는 허용(Approve) 버튼. 오늘은 대부분 승인하면 됩니다.

로그인 / 플랜

워크숍에 따라 다릅니다 — 6/20은 학교 'DCU Medicine RISE' 팀 플랜(@cu.ac.kr), 미니워크숍은 주최측 안내에 따라. 공통: 유료 플랜 권장(무료 계정은 Code 탭 불가), 설치엔 별도 API 키 불필요.

API / API 키Application Programming Interface

프로그램끼리 자동으로 데이터를 주고받는 통로. 따로 쓰는 곳도 있지만 오늘 설치엔 API 키가 필요 없습니다(구독 로그인으로 충분). 구독(OAuth)으로 GPT 등을 무과금 연결하는 법은 → ④ 모델·OAuth.

🛞② 오늘 작업 흐름

파이프라인pipeline

막연한 "논문 써줘"를 작은 단계로 나누고 각 단계에 점검 기준을 붙인 것. 오늘의 핵심 개념.

10단계 lifecycle

논문 한 편을 ①주제 → ②문헌 → ③개요 → ④초안 → ⑤검증 → ⑥비판 → ⑦문체·패키지 → ⑧제출 → ⑨리뷰대응 → ⑩마무리로 한 바퀴 도는 순서.

start-here

"시작하자" 한마디로 위 10단계를 차례로 안내해주는 진입점 스킬. 단계 끝마다 "다음"이라고 하면 넘어갑니다.

운전 모드 / HITLHuman-in-the-loop

각 단계를 "정차(멈춰서 내가 확인)" 할지 "정지 없이 진행" 할지 내가 고르는 것. 사람이 운전대를 쥔다는 뜻.

정차 / Self-Gate

중요한 단계에서 멈춰 저자가 [승인/수정/계속]을 정하는 지점. 오늘 표준 모드는 ①⑤⑥⑩에서 정차.

안전선(floor)

어떤 모드에서도 사람이 반드시 확인하는 선 — 약물·IRB·사망률(의학 안전), AI 공개문·저자권·이해상충(출판 윤리), 실제 제출·로그인.

Human Final Gate

실제 저널에 제출하기 직전, 본인이 최종 확인하는 관문(원자료·참고문헌·저자 승인·윤리 등). 오늘 산출물은 여기 통과 전의 "연습본".

주장 · 근거claim · source

"무엇을 말한다(주장)"와 "그 근거 논문"을 섞지 않고 분리하는 것. 할루시네이션을 막는 핵심 습관.

Deep Research(딥리서치)

문헌을 깊게 찾아주는 기능. 오늘은 ②문헌 단계에서 잠깐 Chat 탭으로 쓸 수 있습니다.

🔬③ AI·논문 기초

LLM(대규모 언어모델)Large Language Model

Claude·GPT·Gemini 같은 AI의 본체. 엄청난 글을 학습해 다음 말을 예측하는 방식이라, 그럴듯하지만 틀릴 수 있습니다.

할루시네이션(환각)hallucination

AI가 사실이 아닌 내용을 자신 있게 만들어내는 것. 의학에서는 환자 안전 문제라 반드시 검증해야 합니다.

교차검증

여러 AI나 출처로 같은 내용을 대조해 오류를 잡는 것. 오늘 교직원포털의 여러 AI를 이 용도로 씁니다.

참고문헌 / 인용reference · citation

논문이 근거로 댄 다른 논문들. AI는 없는 논문을 지어내거나 저자·제목을 섞기도 해서 점검이 필요합니다.

DOI · PMID

DOI=논문의 고유 인터넷 주소. PMID=PubMed(의학 논문 DB)의 논문 번호. 이게 실제로 존재하는지로 위조 인용을 거릅니다.

PubMed · Crossref

논문 정보를 조회하는 공개 데이터베이스. 참고문헌 점검 도구가 여기에 물어봐 진위를 확인합니다(가입·키 불필요).

철회 논문retracted

오류·부정으로 학계에서 취소된 논문. 모르고 근거로 인용하면 큰 문제라 자동으로 걸러줍니다.

desk reject(데스크 리젝)

심사에 들어가기도 전에 편집장이 곧바로 반려하는 것. 형식·범위·참고문헌 오류가 흔한 원인.

투고 / cover letter / AI 공개문

투고=저널에 논문 제출. cover letter=편집장에게 보내는 소개 편지. AI 공개문=AI를 어떻게 썼는지 밝히는 글(요즘 저널 필수).

IRB · COI · ORCID

IRB=연구윤리심의. COI=이해상충(이해관계 충돌) 공개. ORCID=연구자 고유 ID 번호.

🔌④ 더 알아두면 — 모델 연결·OAuth·이미지 (워크숍 너머·관심자용)

OAuth(오어쓰)

비밀번호를 직접 넘기지 않고 "이 앱이 내 계정을 써도 됩니다"라고 안전하게 허용하는 로그인 방식("구글로 로그인" 버튼이 그 예). 오늘 팀 플랜 로그인도 이 방식.

API 과금 vs 구독(OAuth)

AI를 프로그램에서 부를 때 — API 키는 쓴 만큼 돈을 냄(종량제), 구독 OAuth는 이미 낸 월 구독으로 추가 과금 없이 씀.

brain(두뇌) 모델

에이전트가 "생각"에 쓰는 LLM. 작업마다 강점이 달라 골라 씁니다. 과금형(API) = Claude·Gemini(토큰당 과금) · 구독 OAuth형 = GPT·Grok(구독으로 추가 과금 없이).

GPT를 OAuth로 연결

ChatGPT 구독자가 OAuth로 연결하면 API 종량과금 없이 GPT를 Claude 안에서 사용 — GPT와 상의(교차 자문), 이미지 생성 등을 추가 비용 없이.

GPT-image-2

GPT의 이미지 생성 모델. OAuth 연결 시 그림(인포그래픽·피규어 등)을 추가 과금 없이 만들 수 있음.

Grok(그록)

xAI의 LLM. GPT처럼 구독 OAuth로 연결해 무과금 사용 가능.

교차 자문 (3AI)

같은 질문을 Claude·GPT·Gemini 등 여러 모델에 던져 대조 — 한 모델의 오류·할루시네이션을 잡습니다(③ 교차검증을 모델 단위로).

🤖⑤ 더 알아두면 — 비서·에이전트로 키우기 · 세션 관리 (워크숍 너머)

에이전트(agent)

단계를 일일이 지시하지 않아도, 목표만 주면 스스로 도구를 골라 여러 단계를 수행하는 AI.

헤르메스(Hermes Agent)

Nous Research의 오픈소스 자율 비서 에이전트. PC/서버에 상주하며 영속 기억·예약(cron) 실행으로 "○○ 알아봐줘"를 스스로 처리합니다. 모델 무관(Claude 포함), 스킬·연동을 붙일수록 유능해집니다.

K-skills

한국 사용자에게 유용한 스킬 모음(GitHub 공개). 헤르메스에게 시켜 필요한 것만 골라 설치 — 예: 기차표 예약·대중교통·날씨 안내. 비서로서 충실도가 올라감.

Gmail · 구글 캘린더 연동

헤르메스에 이메일·일정을 연결하면 "메일 정리·일정 잡기" 같은 비서 역할이 한층 커집니다. (연동의 기술 통로 = MCP.)

MCPModel Context Protocol

Gmail·캘린더·DB 같은 외부 서비스를 AI에 안전하게 연결하는 표준 통로(연동의 기술 이름).

마크다운 · 위키 · 옵시디언

마크다운= #, - 같은 기호로 제목·목록을 적는 가벼운 서식. 위키= 문서가 링크로 엮인 지식베이스. Obsidian(옵시디언)= 마크다운 노트를 링크로 엮는 인기 지식관리 앱.

LLM wiki

Karpathy가 공개한 패턴 — 폴더+마크다운으로 AI가 쌓을수록 똑똑해지는 지식베이스(매번 새로 찾는 RAG가 아니라 index.md(목차)+log.md(기록)로 누적). "Code에 붙여넣어 쓰라"는 공개 gist 그대로 시키면 구조까지 잡아줌. 예: 친구별 노트(/people)에 자녀 이름·나이를 적어두면 "7월 생일?" "○○ 자녀 나이?"에 바로 답 — 읽은 논문 정리에도. (내 기기에서 개인 노트로)

토큰(token)

AI가 글을 처리하는 단위(대략 단어 조각). 과금·기억량의 기준.

컨텍스트 윈도우context window

AI가 한 대화에서 한 번에 기억하는 양(토큰). 오늘 모델은 최대 100만(1M) 토큰. 단 차면 찰수록 성능이 떨어지고 사용량 한도도 빨리 먹어, 길어지면 새 세션에서 이어가는 게 좋습니다. → ⑥ 사용량 한도

핸드오프(handoff)

지금까지 작업을 정리한 인수인계 메모(MD) + 새 세션 첫 프롬프트를 만들어, 다음 세션이 끊김 없이 이어가게 하는 것. 자주 하므로 스킬로 만들어 두면 "핸드오프 해줘" 한마디로 됨.

스킬로 만들기

반복 작업(핸드오프·교차검증 등)을 스킬로 묶으면 다음부터 한 번에 호출. 이 모든 걸 Claude Code에게 시키면 됩니다 — "이걸 스킬로 만들어줘".

⏱️⑥ 사용량 한도 — 길어지면 끊어가기 (오래 쓰면 꼭 알아둘 것)

⏱️ 한 세션을 너무 길게 끌면 사용량 한도가 빨리 찹니다. 길어지면 /compact하거나 핸드오프로 새 세션에서 이어가세요 — 아래가 그 이유와 방법입니다.
사용량 한도 (5시간 · 주간)

구독 플랜은 5시간마다 리셋되는 한도와 주간 한도가 있습니다. 한도는 처리한 토큰 양에 비례해 쌓여요. 지금 얼마나 썼는지는 /usage로 확인.

왜 갑자기 한도가 차죠?컨텍스트 비대화

Claude Code는 매 턴마다 그때까지의 대화 전체를 다시 처리합니다. 그래서 한 세션이 길어질수록 매 턴이 점점 무거워져 한도를 빠르게 먹습니다. 한도가 걸려 쉰 뒤 같은 거대 세션을 그대로 재개하면 그 큰 맥락이 따라와 몇 턴 만에 또 걸립니다 — "쉬었는데 곧바로 또"가 이것.

/compact

지금까지 대화를 요약해 맥락을 줄이는 명령. 길어진 세션을 이어가되 다음 턴부터 가볍게 만듭니다.

핸드오프로 새 세션

맥락이 차면 핸드오프(→⑤)로 인수인계 메모를 만들고 새 세션에서 이어가면 맥락이 리셋돼 한도 소모가 확 줄어듭니다. 한도가 걸렸을 땐 같은 세션 재개 대신 이 방법.

모델 선택 (Opus · Sonnet · Haiku)

같은 작업도 Opus가 한도를 가장 빨리 먹습니다. 일상 작업은 Sonnet(가볍고 빠름), 정말 어려운 것만 Opus로 — 이것만으로 한도가 크게 늘어납니다. 모델은 앱 또는 /model에서 전환.

🧰⑦ 더 나아가기 — 연동·자동화 도구 (응용반·관심자용)

Connector(커넥터)

Claude의 설정 ▸ Connectors에서 Gmail·구글 캘린더 같은 외부 서비스를 켜는 기능. (연결의 기술 통로가 ⑤의 MCP.) 유료 플랜에서 동작하고, 켠 뒤에도 "물어볼 때만" 작동합니다.

cron · 예약 작업

"매일 아침 9시"처럼 정해진 시각에 자동 실행되는 일정. 커넥터는 물어봐야 동작하므로, "메일 오면 알려줘"처럼 알아서 챙기려면 헤르메스의 cron 예약이 주기적으로 점검해야 합니다.

Claude in Chrome

크롬 확장(유료 플랜 베타). Claude가 사이드패널에서 웹페이지를 읽고·클릭·폼 입력합니다. 안전을 위해 "승인 후 실행" 모드 — 뱅킹 등 민감 사이트는 주의.

Playwright(플레이라이트)

웹브라우저를 자동으로 조작·테스트하는 개발용 오픈소스 도구. Claude Code에 붙여 재현 가능한 자동화에 씁니다(Chrome 확장이 '비서'라면 이건 '스크립트').

Claude Design

claude.ai/design에서 대화로 프로토타입·슬라이드·원페이저를 만드는 도구(유료·Labs). 완성하면 Code로 넘겨 실제 구현까지 이어집니다.

Claude for Office · 애드인 · AppSource

애드인=프로그램에 끼워 기능을 더하는 확장. Microsoft AppSource(애드인 장터)에서 "Claude by Anthropic"을 깔면 Word·Excel·PowerPoint 안에서 Claude를 씁니다. Word는 변경내용 추적으로 제안. ⚠️ 학교·병원 M365는 관리자가 외부 애드인을 막아둘 수 있음.
