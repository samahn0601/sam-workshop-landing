1Gmail·캘린더 연결 ~10분

설정(Settings) ▸ Connectors → Google(Gmail·Calendar) Connect → 계정 인증. (Team/Enterprise는 2중 사전 활성화 필요: ① Claude 조직 Owner가 설정 ▸ Connectors에서 활성화 ② 학교 Google Workspace는 IT가 admin.google.com ▸ 보안 ▸ API 제어에서 Claude를 '신뢰 앱(Trusted)'으로 허용 — 누락 시 "Access blocked" 오류. 데스크톱/웹에서 연결.)

다음 주 내 구글 캘린더 일정을 요약해줘.
내일 오후 3시에 "논문 마감" 일정을 1시간으로 캘린더에 넣어줘. (넣기 전에 확인받고)

Gmail은 검색·요약·초안 작성까지(자동 발송 ❌ — 보내기는 직접). 캘린더는 조회+생성(수정·삭제는 MCP 연결 시).

2투고 메일 추적 자동화 ~15분 · 🅱의 하이라이트

내 Gmail에서 최근 30일간 저널 편집장/투고시스템 메일(editorialmanager·scholarone·manuscriptcentral 도메인, 또는 'decision/revision/manuscript ID/under review' 포함)을 찾아, 논문별 상태를 표로 요약해줘. 리비전 마감 등 기한이 보이면 구글 캘린더에 일정으로 넣어줘(넣기 전 확인). 회신이 필요한 건 자동 발송하지 말고 초안만 만들어줘.
💡 정직하게: 커넥터는 "물어볼 때만" 동작 → "메일 오면 바로 알림"은 실시간 푸시가 아님. 상시 알림은 ④ 헤르메스(cron 예약)나 예약 작업이 이 명령을 주기적으로 돌리는 구조.

5헤르메스(Hermes Agent) — 자율 비서로 확장 개요 ~8분

Nous Research의 오픈소스(MIT) 자율 에이전트 — PC/서버에 상주하며 영속 메모리·스킬 자가생성, 내장 cron으로 예약 작업을 알아서 실행. 왜 핵심: ②의 투고 추적을 "매일 아침 자동 점검 → 알림"처럼 상시화하는 엔진.
🎯 오늘 현장에선 "이런 게 된다"는 개요·데모까지. 아래 브레인·메시징 설정은 외부 계정 발급이 필요해 집에서 천천히 골라서 하시면 됩니다 — 지금 전부 안 해도 됩니다.

설치 — 가장 쉬운 길: 데스크탑 앱

💻 hermes-agent.nousresearch.com에서 공식 데스크탑 설치파일(.exe/.dmg) 받아 실행 → CLI까지 같이 깔리고, 이후 터미널 없이 GUI로 채팅·모델(브레인) 선택·메시징 연결·세션 관리·예약. CLI·데스크탑·웹 대시보드(hermes dashboard)가 상태 공유 — 아래 설정도 데스크탑 앱이면 클릭으로. ※ 제공 형태·시점은 공식 사이트 기준.

또는 — Code에 시켜 설치(터미널 명령 불필요):

헤르메스(Hermes Agent, github.com/NousResearch/hermes-agent)를 내 컴퓨터에 설치해줘. 공식 사이트 hermes-agent.nousresearch.com에서 내 OS(Windows/맥)에 맞는 설치 방법을 찾아 진행하고, 끝나면 첫 설정(hermes setup)까지 안내해줘. 시스템을 바꾸는 명령은 실행 전에 보여주고 확인받아.

→ Claude가 공식 명령 실행(권한·로그인은 직접) → hermes setup. 직접: Win iex (irm https://hermes-agent.nousresearch.com/install.ps1) · mac/Linux/WSL2 curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

⚠️ 설치 스크립트는 원격 실행이니 공식 도메인 확인 후. 공식: docs · github

🧠 브레인(두뇌) 모델 — 카드 없이

💡 당장은 'NVIDIA NIM + Hermes 3' 하나면 충분 — build.nvidia.com에서 무료 키 받아 hermes setup → NVIDIA → Hermes 3 선택. (Hermes 3 = tool-calling 특화라 비서에 딱.) 나머지는 상황별 대안이니 안 봐도 됩니다.
브레인은 tool-calling + 64K↑ 컨텍스트 모델이어야 합니다(아무 모델이나 X). 상황별:

• ChatGPT 구독자 → OpenAI Codex: hermes setup → 브라우저 코드 인증. API 키 불요·기존 구독으로.
• OpenRouter 무료: openrouter.ai 키 → tool-calling 되는 :free 모델(예: gpt-oss·qwen3-coder, 변동). 무료 일일 약 50회·20회/분(공식 기준 · $10+ 1회 충전 시 1,000회/일).
• Claude: 브레인으로 쓸 수는 있으나 개인 구독만으론 제약(API 키=종량과금 / 구독 OAuth는 상위 플랜·크레딧 필요·정책 변동) → 무과금·즉시는 위 NIM·OpenRouter·Codex 권장. (정확한 현행 요건은 Anthropic 확인)

프롬프트로 한 번에(추천 NIM 경로):

📱 어디서나 대화 — 텔레그램·디스코드·슬랙

헤르메스는 메시징 게이트웨이로 텔레그램·디스코드·슬랙(·WhatsApp)에 붙습니다 → 폰에서 내 비서에게 말 걸고 알림 받기. 한 백그라운드 프로세스가 세션·cron·음성까지 처리. "한 번에 받는 다운로드 링크"는 없고 봇 토큰 방식입니다(아래 3단계).

텔레그램 — 한 탭으로 이동:
① t.me/BotFather → /newbot → 봇 토큰 복사
② t.me/userinfobot → 내 숫자 ID 확인
③ hermes gateway setup → Telegram 선택 → 토큰·ID 붙여넣기 → hermes gateway로 시작

프롬프트로 한 번에(Code가 안내):

헤르메스를 텔레그램에 연결하고 싶어. ① BotFather(t.me/BotFather)로 봇 만들어 토큰 받기, ② userinfobot(t.me/userinfobot)으로 내 숫자 ID 확인, ③ hermes gateway setup으로 텔레그램에 '내 ID만' 허용되게 토큰·ID 설정하고 게이트웨이 시작까지 — 순서대로 안내해줘. 봇 토큰은 비밀이니 다시 출력하거나 기록에 남기지 말고.
💡 디스코드·슬랙도 같은 hermes gateway setup(각 봇 토큰·초대). ⚠️ 봇 토큰=비밀번호 — 공유·커밋 금지·허용 사용자에 본인 ID만. 외부 계정(NIM 키·텔레그램 봇)은 발급에 시간이 걸리니 집에서 미리. 실기기 1회 확인 권장.
